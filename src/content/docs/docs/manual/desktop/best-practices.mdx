# 最佳实践

---

## 一、怎么描述任务

**核心原则：告诉它你要什么，而不是让它怎么做。** OpenDesk 会自己规划执行路径。

| 要点 | ✅ 推荐 | ❌ 避免 |
|------|---------|---------|
| 说清目标 | 「生成含图表的数据分析报告」 | 「读文件、分析数据、画图、写报告」 |
| 给出约束 | 「Markdown 格式，500-800 字，技术文档风格」 | 没有任何格式或范围要求 |
| 一个任务一件事 | 任务 1：分析 CSV → 任务 2：生成报告 | 「分析 CSV 并发邮件通知团队」 |

## 二、提供上下文

| 方式 | 示例 |
|------|------|
| **上传文件** | 上传 CSV 做分析、上传代码做审查、上传模板做内容生成 |
| **指定工作目录** | 「分析 /path/to/project 目录下的代码架构」 |
| **引用已有内容** | 「基于上次分析报告的结论，补充竞品分析章节」 |
| **补充背景信息** | 技术栈是什么、项目做什么用、有什么要求等 |

**背景信息示例**：
```
帮我写一个 API 接口：
- 项目：电商后台管理系统
- 技术栈：Node.js + Express + MongoDB
- 用途：商品库存管理
- 要求：支持增删改查、批量操作
```

## 三、高效使用

### 自然语言优先

直接说需求就好，不需要知道底层叫什么工具或技能：

- ✅ 「查看今天的邮件」「添加明天 3 点的会议」「搜索笔记里关于 AI 的内容」
- ❌ 不需要手动调用任何 API 或指定工具名

### 分阶段推进（复杂/高风险任务推荐）

```
阶段1：「分析代码，列出需要重构的问题」  → 你确认问题列表
阶段2：「按优先级重构前3个问题」       → 你确认重构结果
阶段3：「运行测试验证效果」             → 你确认测试结果
```

### 大任务自动拆解

复杂任务 OpenDesk 会自动拆分步骤：

```
你：「分析大型项目并生成文档」
OpenDesk 自动：
├── 步骤1：探索项目结构
├── 步骤2：分析核心模块
└── 步骤3：整合为架构文档输出
```

## 四、追问与迭代

任务完成后可以直接追加需求，效率最高：

| 要点 | 说明 |
|------|------|
| **基于已有内容调整** | 无需重新描述背景，直接说改什么 |
| **指明具体位置** | 「在第三章添加性能分析」「修改那个认证模块的错误处理」 |
| **微调优于重来** | 小改动→追问迭代；大方向变→重新描述 |

| 方式 | 效率 | 适用场景 |
|------|------|---------|
| 追问迭代 | 高 | 微调、补充、格式修改 |
| 重新描述 | 低 | 全新方向、不同目标 |

## 五、安全与风险控制

| 要点 | 建议 |
|------|------|
| **风险操作会主动确认** | 删除文件/目录、覆盖文件、发送邮件等操作——都会让你先确认 |
| **提前备份重要文件** | 「先备份原文件再修改配置」 |
| **明确操作范围** | 「清理临时文件但不要删除日志文件」「重构不改变对外接口」 |

## 六、效率技巧

| 技巧 | 示例 |
|------|------|
| **建立模板** | 固定格式的周报/审查报告/会议纪要——描述一次模板后可重复使用 |
| **利用长期记忆** | 让它记住你的偏好（语言风格、文档格式、编码规范），跨会话保持 |
| **并行多任务** | 同时创建多个独立任务并行执行，省时间 |
| **善用追问** | 在已有结果上迭代调整，比每次从头描述高效得多 |

## 七、常见误区

| 误区 | 怎么改进 |
|------|---------|
| 描述太模糊（「帮我处理一下文件」） | 具体化：「整理 Downloads 目录，按类型分到子文件夹」 |
| 要求它做不到的事 | 先了解能力边界：无法操作需要登录且无 API 的网站、无法处理二进制原始数据 |
| 不看执行过程直接用结果 | 浏览一下执行步骤，确认理解正确再采纳结果 |
| 对风险操作不检查 | 重要任务分阶段确认，逐一审核变更内容 |

## 速查卡

| 维度 | ✅ 推荐 | ❌ 避免 |
|------|---------|--------|
| **描述任务** | 说目标 + 给约束 | 列执行步骤 / 模糊需求 |
| **给上下文** | 上传文件 + 指定目录 + 补背景 | 让它猜你的意图 |
| **交互方式** | 自然语言描述需求 | 手动调用底层接口 |
| **任务管理** | 大任务分阶段确认 | 一个大任务丢下去不管 |
| **迭代方式** | 追问微调已有结果 | 每次从头描述一遍 |
| **安全意识** | 先备份 + 明确范围 + 确认操作 | 跳过确认直接执行 |
| **效率习惯** | 建立模板 + 利用记忆 + 并行执行 | 重复输入相同描述 |

## 场景快速参考

| 场景 | 关键要点 |
|------|---------|
| 生成文档 | 提供模板 + 格式要求 + 字数限制 |
| 代码重构 | 分阶段确认 + 指定范围 + 跑测试验证 |
| 数据分析 | 上传文件 + 指定输出格式 + 追问补充维度 |
| 批量处理 | 指定目录 + 设置约束 + 确认操作范围 |
| 邮件处理 | 明确操作类型 + 确认发送内容 |
| 日程安排 | 提供时间 + 参与人 + 提醒要求 |
| 联网搜索 | 说清关键词 + 结果数量要求 |
