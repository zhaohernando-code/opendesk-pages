# 功能指南

OpenDesk 的能力覆盖文件处理、信息获取、开发辅助、日常办公等多个方面。以下介绍它能帮你做什么，以及怎么用效果最好。

---

## 文件管理

| 你想做什么 | 怎么做 | 适合场景 |
|-----------|-------|---------|
| 读取文件内容 | 直接说「读一下 xxx 文件」 | 查看代码、配置、数据、日志 |
| 创建或写入文件 | 「帮我创建一个 xxx 文件，内容是…」 | 写文档、生成配置、出报告 |
| 修改文件中的内容 | 「把 xxx 文件中的 aaa 改成 bbb」 | 改代码、换配置项、批量替换 |
| 搜索特定文件 | 「找一下项目中所有 .ts 文件」 | 统计文件、按类型查找 |
| 在文件中搜索内容 | 「搜索包含 用户认证 的代码」 | 找函数定义、查引用位置 |
| 用自然语言找文件 | 「找和认证相关的文件」 | 不确定文件名时按语义查找 |

> **提示**：指定工作目录后，OpenDesk 仅在该目录内操作，不会影响其他位置的文件。

## 信息获取

| 你想做什么 | 示例 |
|-----------|------|
| 联网搜索 | 「搜一下最新的 React 19 新特性」 |
| 学术论文检索 | 「搜 Transformer 相关的最新论文（arXiv）」 |
| 抓取网页内容 | 「帮我读取这个网页的文章内容」（提供 URL） |

## 开发辅助

| 你能做什么 | 示例 |
|-----------|------|
| 运行命令和脚本 | 「在项目目录下运行 npm test」「编译这个 Go 项目」 |
| 理解代码库 | 「帮我看一下这个项目的架构」「这个函数是做什么的」 |
| 写代码和改代码 | 「写一个用户登录接口」「重构 auth 模块，统一错误处理」 |
| 诊断问题 | 「这个 Bug 是什么原因引起的」「为什么测试失败了」 |

## 日程与待办

| 能力 | 说明 |
|------|------|
| 查看时间 | 获取当前时间、日期、时区等信息 |
| 待办管理 | 规划复杂任务步骤、追踪进度——OpenDesk 会自动拆分任务并逐项推进 |

## 任务与记忆

| 能力 | 说明 |
|------|------|
| 多任务并行 | 同时创建多个任务，各自独立执行，互不干扰 |
| 任务历史 | 回顾之前做过什么、查看执行过程和结果 |
| 长期记忆 | 记住你的偏好（如编码风格、文档格式要求）、重要决策、项目信息，跨会话保持 |

## 交互方式

### 选择与确认

当 OpenDesk 需要你的决策时，会给出选项供你选择：

- 选择技术方案（「用 A 方案还是 B 方案？」）
- 确认风险操作（「即将删除以下 3 个文件，是否继续？」）
- 澄清需求歧义（「你指的是 A 还是 B？」）

### 追问与迭代

任务完成后可以直接追加需求，无需重复背景：

- 「再加一个饼图展示占比」
- 「把字体调大一点」
- 「导出为 PDF」

---

## 使用建议

| 建议 | 说明 |
|------|------|
| **描述目标，不描述步骤** | 说「生成分析报告」，不说「先读文件再画图再写报告」 |
| **提供上下文** | 上传文件、指定目录、补充格式要求，结果更准确 |
| **分阶段确认** | 复杂任务可以先让它出方案，确认后再动手 |
| **善用追问** | 基于已有结果微调比重新描述更高效 |
